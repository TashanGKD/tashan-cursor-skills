## 一、人层设计（Human Layer）

### 1.1 解决什么问题？

```
用户让 AI 调研某主题，AI 只在对话里口头回答，不持久化；
或 AI 随机在错误位置创建文件（_内部总控/知识库/），无标准格式、无图表、未注册。
目标：任何网络调研请求 → 自动产出结构化图文文档 → 存对位置 → 注册到认知结构。
```

### 1.2 触发条件（人的视角）

```
典型触发语句：
- 「帮我调研 X」「调研一下 X」「系统调研 X」
- 「研究一下 X」「帮我系统了解 X」「深度研究 X」

触发场景：
- 用户想对某个不熟悉的领域/框架/概念建立系统认知
- 该内容来自外部（非郑总个人思考），需要网络搜索

明确不触发的情况：
- 「搜索一下这个 API 怎么用」（临时查找，不是系统调研）
- 「帮我找一下 XXX 的错误原因」（调试，不是调研）
- 「这个概念是什么意思」（单点解释，不是系统调研）
```

### 1.3 成功标准（人的视角）

```
产出物：
  - 完整 Markdown 文档（结构化：摘要+分章节+来源标注）
  - Mermaid 结构图（V1 写入，保留）
  - qwen-image 生成图（自审后嵌入，与 Mermaid 共存）
  - REF-EXT 头部标注
  - 保存到正确维度的知识库/目录

质量判断标准：
  - 打开文档能系统了解该主题（不需要再问 AI）
  - 有图有表，结构清晰
  - 分类清单和知识图谱里有对应条目
```

### 1.4 MVP 边界

```
✅ 做：
  - 6个 L1 维度的自动判断（含无法判断时询问）
  - 详细 Markdown + Mermaid（V1）→ 自审 → qwen-image 嵌入
  - REF-EXT 头部 + 文件保存
  - 自动注册（分类清单 + 知识图谱 + 系统日志）

❌ 不做：
  - 郑总个人思考的输出（那是 L2 碎片，属于 cognitive-capture-fragment）
  - 发布/分享这个文档（输出只到知识库，不做 HTML 排版）
  - 多文档对比分析（那是 architecture-gap-mapper）
```

## 二、Agent 层设计（Agent Layer）

### 2.1 触发条件（AI 视角）

```
触发条件（精确）：
  IF 用户输入包含以下任一关键词：
    「调研」「系统调研」「深度调研」「研究一下」「帮我系统了解」「帮我深入了解」
  AND 后面跟随一个明确的主题词（非代词）
  THEN → 触发本 Skill

不触发（负条件）：
  - 「搜索一下」「查一下」「找一下」→ 临时查找，不触发
  - 「这个是什么意思」→ 单点解释，不触发
  - 「帮我调试/排查」→ 不触发
  - 无主题词（如「调研一下」后面没有具体内容）→ 询问主题后触发

与其他 Skill 的优先级：
  - research-output 触发后，不被 wechat-article-writer/general-article-writer 抢占
  - 如果用户说「调研 X 然后写一篇文章」→ 先执行 research-output，文章写作另起
```

### 2.2 执行步骤（精确序列）

```
Step 0  维度判断
        输入：调研主题
        操作：按以下规则映射到 L1 维度
          技术框架/开源工具/架构/编程/系统设计 → 技术架构思维维度
          认知心理学/脑科学/学习/记忆/问答机制 → 认知科学基础维度
          产品设计/市场/竞品/商业模式/用户研究 → 产品理论维度
          写作/发布/内容创作/配图/工具使用     → 待建维度
          研究方法/学术规范/论文/数字人          → 研究范式维度
          组织设计/团队协作/AI组织/社区          → 组织理论维度
        输出：目标路径 `_内部总控/认知结构/L1_系统性文档/[维度]/知识库/`
        失败时：输出「这个主题属于哪个维度？[6个选项]」等待用户回答

Step 1  网络调研 + 撰写 V1 草稿（含 Mermaid）
        输入：调研主题 + 维度（确定背景视角）
        操作：
          - 网络搜索获取核心信息
          - 撰写结构化 Markdown：
              摘要（核心结论3-5条）
              目录
              各章节（含来源标注）
              参考来源列表
          - 在适合的位置插入 Mermaid 代码块：
              流程图（flowchart）→ 流程/决策/步骤
              类图/状态图       → 系统架构/状态转换
              时序图             → 交互时序
              Mermaid 用于结构表达，不替代数据图表
        输出：完整 V1 Markdown（含 Mermaid 代码块）
        失败时：若某章节无可信来源，标注「[需补充：缺乏可信来源]」，不编造

Step 2  自审：识别 qwen-image 画图位置
        输入：V1 草稿
        操作：
          Read: _内部总控/认知结构/L1_系统性文档/认知科学基础维度/知识库/全场景绘图指南.md
          按以下标准识别适合 qwen-image 的位置：
            - 需要展示分层架构/概念框架图 → 生成概念框架图
            - 需要展示对比/矩阵 → 生成对比图
            - 需要展示数据/统计 → 生成数据图
            - 纯文字说明/列表/代码 → 不画图
            - 已有 Mermaid 的位置 → 在 Mermaid 代码块之后插入 qwen 图（两者共存）
        输出：标注了画图位置的列表（位置 + 图类型 + 提示词草稿）
        失败时：绘图指南文件不存在 → 跳过 Step 2，直接到 Step 3 保存文字版

Step 3  调用 qwen-image 生成图片并嵌入
        输入：Step 2 的画图位置列表
        操作（对每个位置）：
          调用 qwen-image API：
            API_KEY = 'sk-68b70d6863b94c299ecd27e9d49b41ba'
            ENDPOINT = 'https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation'
          提示词格式：「简洁专业的[图类型]，白色背景，[具体内容描述]」
          图片以 base64 嵌入 Markdown：![图描述](data:image/png;base64,...)
          Mermaid 代码块保留，qwen 图插入其下方（用注释分隔）：
            ```mermaid
            [原 Mermaid 代码]
            ```
            > 📊 可视化版本：
            ![可视化](data:image/png;base64,...)
        输出：完整 V2 Markdown（Mermaid + qwen 图共存）
        失败时：单张图生成失败 → 跳过该图，继续其他图；全部失败 → 使用 V1 文字版

Step 4  保存文件 + 添加 REF-EXT 头部
        输入：V2 Markdown + 维度路径 + 调研日期
        文件命名规则：`[主题关键词（中文/英文）]_调研_YYYYMMDD.md`
        写入路径：`_内部总控/认知结构/L1_系统性文档/[维度]/知识库/[文件名]`
        在文档开头插入 REF-EXT 头部：
          > 文档类型：**REF-EXT**（外部参考知识）
          > 调研日期：YYYY-MM-DD
          > 调研目的：[从用户请求提取]
          > 来源说明：AI网络调研产出
          > 注意：本文档为外部知识汇编，不是郑总个人思考产物，不进入认知结构 L1/L2/L3
          > 分类登记：文档分类清单.md 区域十二
        输出：写入文件成功
        失败时：告知用户路径，建议手动保存

Step 5  自动注册（三处）
        ① 文档分类清单.md 区域十二：追加一行
           | [路径] | REF-EXT | [维度] | ★ CURRENT | [一句话描述] |
        ② 知识图谱_正式文档.md REF-EXT 区段：追加一行
           | [文档ID] | [简称] | [维度] | [核心内容一句话] | [知识库路径] | [关联L1文档] |
        ③ 系统日志.md：追加一条
           [LOG-YYYYMMDD-NN] research-output | 调研：[主题] → [维度]/知识库/[文件名]
        输出：三处注册完成，告知用户文件保存位置和知识图谱 ID
        失败时：若任一注册失败 → 告知用户哪步失败，建议手动补注册
```

### 2.3 边界条件

```
情况A：主题跨多个维度（如「AI 产品中的认知科学应用」）
  → 默认选更具体的维度（认知科学基础维度）
  → 告知用户「已存到认知科学基础维度，若需移动请告知」

情况B：qwen API 全部失败
  → 保存纯文字+Mermaid版本，不阻塞流程

情况C：用户打断（只想要口头回答，不想要文档）
  → 若用户说「不用存文件/只是问问」→ 跳过 Step 4/5，只输出内容

情况D：同主题已有调研文档
  → 检测到 知识库/ 下有同名文件 → 询问用户「已有[文件名]，是更新还是新建？」
```

### 2.4 依赖声明

```
依赖的外部资源：
- qwen-image API（Step 3）
- 网络搜索能力（Step 1）
- 全场景绘图指南.md（Step 2 读取）

依赖的认知结构系统：
- 维护协议_自洽规范.md 的 REF-EXT 规则（Step 4 遵循）
- 文档分类清单.md（Step 5 写入）
- 知识图谱_正式文档.md（Step 5 写入）
- L3_原始记录/系统日志.md（Step 5 写入）
- L1_系统性文档/[维度]/知识库/ 目录（Step 4 写入文件）

与其他 Skill 的关系：
- 不调用 wechat-article-writer / general-article-writer
- 不调用 article-proofreading（调研文档不走写作审稿流程）
- 不触发 auto-experience-hook 的信号B（这是规范行为，不是错误）
```

### 2.5 常见失败模式

```
失败模式1：维度判断错误（常见：技术+产品交叉主题判断为错误维度）
  根本原因：映射规则不够细
  预防：Step 0 映射规则明确，且输出维度判断结果让用户确认（可选）

失败模式2：跳过 Step 5 注册（认为保存文件后「已完成」）
  根本原因：Step 4 保存成功 = 心理完成感
  预防：Step 5 放在同一连续序列，不允许在 Step 4 后输出「完成」信号

失败模式3：V1 和 V2 文档混乱（覆盖了 V1 Mermaid）
  根本原因：Step 3 直接写文件时删除了 Mermaid
  预防：Step 3 明确「Mermaid 代码块保留，qwen 图插入其下方」

失败模式4：知识图谱注册时关联 L1 文档填写错误
  根本原因：AI 猜测关联关系
  预防：若无明确关联 → 关联列填「—」，不猜测

回退动作：
- 若 Step 4 保存失败 → 在对话中输出完整文档内容，等用户手动保存
- 若 Step 5 注册失败 → 告知缺失的注册项，用户可后续补做
```

### 2.6 强绑定 Rule

```
强绑定：
- R2 NO_FABRICATION：调研内容必须来自实际网络信息，不编造
- R1 EVIDENCE_FIRST：来源标注明确，区分「已查到」与「推断」

推荐注入：
- R6 ARTIFACT_FIRST：输出为正式 Markdown 文件，不只是对话回答
```

## 三、人层 vs Agent 层张力分析

```
人层隐含假设 → Agent 层需显式告知：
- 「调研」隐含「保存到文件」→ 已在 Step 4/5 明确
- 「图」隐含「两种图共存，Mermaid 不删」→ 已在 Step 3 明确
- 「注册」隐含「三处同步」→ 已在 Step 5 列出三处

AI 可能歧义的点：
- 「研究一下」可能被理解为「解释一下」→ 触发条件加了「主题词」要求
- 「嵌入图片」可能被理解为引用本地路径 → Step 3 明确 base64 嵌入
- Step 4 和 Step 5 可能被分开执行 → 说明两者是连续步骤
```
