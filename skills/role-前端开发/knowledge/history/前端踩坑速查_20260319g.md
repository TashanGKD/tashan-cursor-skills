# 前端踩坑速查

> 来源：历史项目真实踩坑。每次遇到新问题，追加到本表。

## 踩坑速查表

| # | 症状 | 根因 | 修复方案 |
|---|---|---|---|
| FE-01 | API 请求静默失败，控制台报 "Unexpected end of JSON input" | vite proxy 端口与后端 BACKEND_PORT 不一致 | 检查 vite.config.ts 中 proxy target 端口，与 .env 中 BACKEND_PORT 完全一致 |
| FE-02 | 验证码框不显示，用户无法注册 | `<input>` 藏在 `codeSent && (...)` 条件渲染后 | 始终渲染验证码框，只用 `disabled` 控制是否可交互 |
| FE-03 | 业务 API 调用报错，import 找不到 | 业务 API 混入 auth.ts 后，重构时被删除 | **业务 API 必须放独立文件**（如 orgApi.ts），不能混入 auth.ts |
| FE-04 | 登录后看不到应有的引导（如 AI 引导页），直接跳到聊天 | 根路由 `/` 固定跳转 `/chat`，未判断用户当前状态 | 实现 `HomeRedirect` 组件，检查用户是否有活跃组织，按状态跳转 |
| FE-05 | dev_code 一闪而过，用户来不及复制 | 没有给 dev_code 展示设置足够时间 | dev_code 展示计时器设为 15 秒，显示倒计时 |
| FE-06 | SSE 连接建立成功但收不到内容 | 用了 EventSource（不支持自定义请求头，无法带 Authorization） | 改用 fetch + ReadableStream 方式消费 SSE |
| FE-07 | TypeScript 类型错误导致构建失败 | API 返回类型与前端定义不一致 | 在 `src/types/` 维护与后端 Pydantic Schema 对应的 TypeScript 类型定义 |
| FE-08 | 页面加载慢，白屏时间长 | 没有代码分割，所有组件打进一个 bundle | 使用 `React.lazy + Suspense` 做路由级懒加载 |
| FE-09 | 浏览器访问 `http://localhost:3000` 报 Error -102 (ERR_CONNECTION_REFUSED) | Node 18+ 在 Windows 上默认将 `localhost` 绑定到 `::1`（IPv6），Chrome 先尝试 IPv4 `127.0.0.1` → TCP 被拒绝 | 在 `vite.config.ts` 的 `server` 节加 `host: '127.0.0.1'` 强制 IPv4 绑定；或直接访问 `http://127.0.0.1:3000/` |
| FE-10 | 浏览器控制台报 `ReferenceError: xxx is not defined`，功能静默失效 | 新增了对某函数的调用，但没有更新 `import` 语句。TypeScript 编写时不报错，运行时才暴露。常见场景：新增 SSE 事件处理、新增 API 调用、重构时移动了函数所在文件 | **写完任何新的函数调用后立即检查 import**：`Ctrl+F` 搜索函数名，确认 import 行存在。自测清单必须包含「新增调用的函数 import 已更新」|
| FE-11 | Vite 报 `[PARSE_ERROR] Unexpected token`，指向一行孤立的 `/>` 或 `)}` | 使用 StrReplace 删除代码时，锚点范围不足，只删除了代码块的开头/中间，结尾的 JSX 关闭符（`/>` `)}` `</div>`）被遗留为游离碎片 | **StrReplace 删除代码时，锚点必须覆盖完整的代码块**：从开头到最后一个关闭符（含 `)`）。删除后立即在浏览器确认无语法错误 |

| FE-11 | 流式输出时用户向上翻页，界面立即跳回底部，无法查看历史消息 | `useEffect(() => { msgEndRef.current?.scrollIntoView() }, [messages])` 无条件在每次消息更新时滚动到底部，完全忽略用户是否正在向上翻页 | 用 `ref`（非 `state`）追踪用户是否在底部，仅在底部时自动滚动；向上翻时显示浮动「↓ 新消息」按钮；用户发送新消息时重置为底部锁定（详见下方完整代码）|

## FE-11 详细方案：流式输出智能滚动锁定

**适用场景**：所有含流式输出（SSE/流式响应）的聊天界面。

```typescript
// 1. 用 ref 追踪用户是否在底部（不触发重渲染）
const isAtBottomRef = useRef(true)
const scrollContainerRef = useRef<HTMLDivElement>(null)
const msgEndRef = useRef<HTMLDivElement>(null)
const [showScrollBtn, setShowScrollBtn] = useState(false)

// 2. 监听滚动事件，实时更新状态
const handleScroll = useCallback(() => {
  const el = scrollContainerRef.current
  if (!el) return
  const atBottom = el.scrollHeight - el.scrollTop - el.clientHeight < 80  // 80px 阈值容错
  isAtBottomRef.current = atBottom
  setShowScrollBtn(!atBottom)  // 控制「↓ 新消息」按钮显示
}, [])

// 3. 只在底部时才自动滚动
useEffect(() => {
  if (isAtBottomRef.current) {
    msgEndRef.current?.scrollIntoView({ behavior: 'smooth' })
  }
}, [messages])

// 4. 「回到底部」按钮：点击后重新锁定自动滚动
const scrollToBottom = useCallback(() => {
  msgEndRef.current?.scrollIntoView({ behavior: 'smooth' })
  isAtBottomRef.current = true
  setShowScrollBtn(false)
}, [])

// 5. 用户发送新消息时重置为底部锁定
const handleSend = () => {
  isAtBottomRef.current = true
  setShowScrollBtn(false)
  // ... 发送消息逻辑
}

// JSX 结构
// <div ref={scrollContainerRef} onScroll={handleScroll} style={{ overflowY: 'auto' }}>
//   {messages.map(...)}
//   <div ref={msgEndRef} />
// </div>
// {showScrollBtn && <button onClick={scrollToBottom}>↓ 新消息</button>}
```

**关键点**：
- 用 `ref` 而非 `state` 追踪滚动位置，避免不必要的重渲染
- 80px 的阈值容错，不需要精确到 0
- `scrollContainerRef` 挂在可滚动容器上，`msgEndRef` 挂在消息列表末尾的空 div

## 账号系统完整规范

### 注册流程（按顺序实现，不可跳过）

```
1. 手机号输入框（始终显示）
2. 发送验证码按钮（未填手机号时 disabled）
3. 验证码输入框（始终显示，发送前 disabled，发送后可输入）
4. 密码输入框
5. 用户名输入框
6. 注册按钮

API 调用顺序：
POST /api/auth/send-code  { phone }
POST /api/auth/register   { phone, code, password, username }
→ 注意参数顺序：(phone, code, password, username)
→ 成功后自动调用 login，获取 JWT Token
→ Token 存入 localStorage
```

### 登录流程

```
POST /api/auth/login  { phone, password }
→ 获取 { access_token, token_type, user }
→ token 存 localStorage
→ 检查用户状态 → 按状态跳转（HomeRedirect）
```

### Token 管理

```typescript
// api/auth.ts
const TOKEN_KEY = 'auth_token';

export const getToken = () => localStorage.getItem(TOKEN_KEY);
export const setToken = (token: string) => localStorage.setItem(TOKEN_KEY, token);
export const removeToken = () => localStorage.removeItem(TOKEN_KEY);

// axios 拦截器自动附加 token
axiosInstance.interceptors.request.use(config => {
  const token = getToken();
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

## Dockerfile 规范（不得修改）

```dockerfile
# 前端构建
FROM node:20-slim AS builder  ← 不用 node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

```nginx
# nginx.conf
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;  ← SPA 路由支持
    }
    
    location /api {
        proxy_pass http://backend:810X;  ← 代理到后端
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 空状态处理（必须实现）

每个列表/内容页面，必须处理三种状态：
1. **加载中**：显示 skeleton 或 spinner
2. **空状态**：显示引导文案 + 行动按钮（不能显示空白）
