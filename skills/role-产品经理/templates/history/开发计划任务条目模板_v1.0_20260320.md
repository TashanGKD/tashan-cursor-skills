# 开发计划任务条目格式规范

> 所有任务条目必须遵循本格式。

---

## 状态符号说明

| 符号 | 含义 | 触发条件 |
|---|---|---|
| ✅ | 功能完成 | 对照产品文档逐条验证，通过关卡C |
| ⚙️ | 架构就绪 | 代码存在但功能未接通执行路径，或未验证 |
| 🔧 | 简化实现 | 有意简化；**必须同时追加"待补充"条目** |
| ⏳ | 待开始 | 未开始 |
| 🔄 | 进行中 | 当前正在处理 |

**核心禁令**：
- `写代码成功 ≠ 功能完成`
- `工具文件存在 ≠ 能力可用`
- `函数名叫 _xxx_with_llm ≠ 真的用了 LLM`（诚实命名原则）

---

## 任务条目标准格式

```markdown
### 闭环X.Y | ⏳ | [任务名称]

**描述**：[1-2句话描述这个任务要做什么]

**完成标准**（标 ✅ 前逐条验证）：
- [ ] [产品文档具体条目1 - 用户可感知行为]
- [ ] [产品文档具体条目2 - 用户可感知行为]
- [ ] [跨层验证：写入的数据能被下游读取]
- [ ] DB验证：实际调用确认写入
- [ ] 核心路径 E2E 测试：[具体路径描述]

**输出接口**：
→ 向 [下游角色/功能] 提供 [什么数据/能力]
→ [其他下游...]

**依赖**：[前置完成的任务，无则写"无"]
```

## 示例任务条目（参考）

```markdown
### 闭环1.2 | ⏳ | 用户注册与登录

**描述**：实现手机+验证码+密码的注册流程，以及 JWT 登录认证。

**完成标准**（标 ✅ 前逐条验证）：
- [ ] 用户可以用手机号发送验证码（接收到真实短信）
- [ ] 验证码输入框始终显示（不藏在 codeSent 条件后）
- [ ] 注册成功后自动登录，跳转到首页
- [ ] 错误情况（验证码错误、已注册手机号）有明确提示
- [ ] dev_code 在响应含有时展示 15 秒
- [ ] DB验证：user 记录确实写入 public.users 表

**输出接口**：
→ 向 [业务功能] 提供 JWT Token 和 user_id
→ 向 [双轨认证] 提供用户认证基础设施

**依赖**：无（P0 基础功能）
```
