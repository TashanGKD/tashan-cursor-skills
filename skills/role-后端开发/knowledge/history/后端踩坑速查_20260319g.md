# 后端踩坑速查

> 来源：历史项目真实踩坑。每次遇到新问题，追加到本表。

---

## 踩坑速查表

| # | 症状 | 根因 | 修复方案 |
|---|---|---|---|
| BE-01 | 所有 API 请求静默失败（无报错，但无响应） | `async with` 用在同步的 postgres_client 上 | 改用 `with get_db_session() as session:` 同步模式 |
| BE-02 | 注册接口返回 422 | register 参数顺序错误 | 按 `(phone, code, password, username)` 顺序 |
| BE-03 | JSONB 字段写入后为空 / None | `:state::jsonb` 语法中 `::jsonb` 被 SQLAlchemy 解析为参数前缀，`state` 参数被静默丢弃 | **改为 `CAST(:state AS jsonb)`**，写完任何 JSONB 写入函数立即测试 |
| BE-04 | `KeyError: 'id'` 在业务代码 | JWT payload 字段不一致：TopicLab 用 `sub`，业务代码期待 `id` | 实现 `_normalize_user()` 统一补全 `id` 和 `sub` |
| BE-05 | 跨项目 schema 引用报错 | `auth.py` 引用 `topiclab.users`，但实际表在 `public` schema | 改为 `public.users`（跨项目统一） |
| BE-06 | SSE 连接成功但客户端永远转圈 | DB 写入失败被静默忽略，SSE generator 没有全局 try/except | **所有 SSE generator 必须有全局 try/except**，异常时输出 error event |
| BE-07 | 函数名说用 LLM，但没有 AI 效果 | `_plan_with_llm()` 内部是硬编码规则，命名欺骗 | 按诚实命名原则重命名为 `_plan_with_rules()`，加 TODO 注释 |
| BE-08 | uvicorn 不断重载，服务不稳定 | 临时调试脚本（如 check_db.py）放在 backend/ 目录下，watchfiles 触发重载 | 临时脚本放在项目根目录或 `/tmp`，不放 uvicorn 监控的代码目录 |
| BE-09 | 多用户数据泄露（用户A看到用户B数据） | SQL 查询没有加 `WHERE user_id = :user_id` 过滤 | 所有涉及用户数据的查询，必须加 user_id 过滤 |
| BE-10 | 环境变量缺失时系统启动成功但运行时崩溃 | 环境变量在第一次用到时才读取，没有启动时校验 | 在 `config/settings.py` 中，所有必须的环境变量用 `os.environ` 读取（无默认值），缺失则启动时 raise |
| BE-11 | 登录/注册返回 500，前端报 `Unexpected token 'I'` | `sessionmaker(autocommit=True)` 在 SQLAlchemy 2.x 中已移除，抛出 `ArgumentError` | 改为 `sessionmaker(autocommit=False)`，context manager 里显式 `commit()`/`rollback()` |
| BE-12 | Docker build 时 pip 报 `--index-url option requires 1 argument` | Dockerfile 中 `ARG PIP_INDEX_URL` 在 `FROM` 之前声明，`FROM` 之后变量值变为空 | **ARG 必须在 FROM 之后重新声明**才能在 RUN 中使用 |

## PostgreSQL 连接规范

### 正确用法

```python
# postgres_client.py（来自 Tashan-TopicLab）
from contextlib import contextmanager
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

engine = create_engine(DATABASE_URL, pool_pre_ping=True)
SessionLocal = sessionmaker(bind=engine)

@contextmanager
def get_db_session():
    session = SessionLocal()
    try:
        yield session
        session.commit()
    except Exception:
        session.rollback()
        raise
    finally:
        session.close()

# 使用方式（同步）
def get_user(user_id: int):
    with get_db_session() as session:
        result = session.execute(
            text("SELECT * FROM public.users WHERE id = :id"),
            {"id": user_id}
        ).fetchone()
        return result
```

### 常见 SQL 注意事项

```python
# JSONB 写入（必须用 CAST 语法）
session.execute(
    text("""
        INSERT INTO t (id, data) 
        VALUES (:id, CAST(:data AS jsonb))
        ON CONFLICT (id) DO UPDATE SET data = CAST(:data AS jsonb)
    """),
    {"id": 1, "data": json.dumps(my_dict)}
)

# 数组写入
session.execute(
    text("INSERT INTO t (tags) VALUES (CAST(:tags AS text[]))"),
    {"tags": "{tag1,tag2}"}
)

# 时间戳（用 UTC）
session.execute(
    text("INSERT INTO t (created_at) VALUES (NOW() AT TIME ZONE 'UTC')"),
    {}
)
```

## 环境变量管理规范

```python
# config/settings.py
import os

class Settings:
    DATABASE_URL: str = os.environ["DATABASE_URL"]  # 无默认值，缺失则报错
    LLM_API_KEY: str = os.environ["LLM_API_KEY"]
    LLM_BASE_URL: str = os.environ.get("LLM_BASE_URL", "https://coding.dashscope.aliyuncs.com/v1")
    LLM_DEFAULT_MODEL: str = os.environ.get("LLM_DEFAULT_MODEL", "qwen3.5-plus")
    JWT_SECRET: str = os.environ["JWT_SECRET"]
    BACKEND_PORT: int = int(os.environ.get("BACKEND_PORT", "8000"))

settings = Settings()
```

## LLM API 规范（阿里云 DashScope）

```python
# LLM API 调用（统一封装）
from openai import OpenAI

client = OpenAI(
    api_key=settings.LLM_API_KEY,
    base_url=settings.LLM_BASE_URL,
)

def call_llm(prompt: str, system: str = "", model: str = None) -> str:
    response = client.chat.completions.create(
        model=model or settings.LLM_DEFAULT_MODEL,
        messages=[
            {"role": "system", "content": system},
            {"role": "user", "content": prompt}
        ]
    )
    return response.choices[0].message.content
```

## Nginx 配置规范（SSE 必须）

```nginx
location /api {
    proxy_pass http://backend:8000;
    
    # SSE 必须的配置
    proxy_buffering off;           # 禁用缓冲，实时推送
    proxy_read_timeout 600s;       # 长连接超时
    proxy_http_version 1.1;        # 支持 chunked transfer
    
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header Connection '';  # 保持连接
}
```
